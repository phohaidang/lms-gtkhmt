BUỔI 6 — ĐỘ HIỆU QUẢ CỦA CHƯƠNG TRÌNH (BIG-O)

Chương 6 của giáo trình · Thời lượng 5 giờ trên lớp. Vai trò trong câu chuyện xuyên suốt: Hệ thống ngân hàng của ta đã chạy được. Nhưng khi triển khai cho 1 triệu khách, chương trình chậm đến mức không dùng được. Buổi này ta học cách đo và dự đoán tốc độ chương trình mà không cần chạy thử trên dữ liệu lớn — đó chính là Big-O.

Trọng tâm đề thi: Chương 6 chiếm 9/40 câu (22.5%) — cao nhất cùng với Chương 5. Đây là buổi cuối khoá, định hình điểm thi.

⚠ Sau buổi này có Kiểm tra giữa kỳ 2 (theo đề cương) — dành 30 phút cuối ôn nhanh Chương 4-6.

I. MỤC TIÊU BUỔI HỌC

Sau buổi này, sinh viên có thể:

  1. Hiểu vì sao "chương trình đúng" chưa đủ — còn cần "chương trình đủ nhanh".
  2. Đếm số phép tính cơ bản trong một đoạn code đơn giản.
  3. Đọc và viết các ký hiệu Big-O thường gặp: O(1), O(log n), O(n), O(n log n), O(n²), O(2ⁿ).
  4. Đối chiếu các thuật toán quen thuộc với độ phức tạp tương ứng.
  5. So sánh hiệu suất cấu trúc dữ liệu cơ bản: list vs dict vs set.
  6. Đo thời gian chạy thực tế bằng module time.
  7. Lựa chọn được giải pháp nhanh hơn cho bài toán đơn giản.

Kết nối CLO: CLO1 (khái niệm), CLO2 (áp dụng), CLO3 (giải vấn đề).

II. KHUNG SCQA CỦA BUỔI

Phần Nội dung
Situation Hệ thống của ta hoạt động ổn với 100 khách. Sếp muốn mở rộng cho 1 triệu khách.
Complication Demo trực tiếp: cùng một đoạn code "tìm kiếm khách hàng" chạy 0.001s với 1000 khách, nhưng mất 5 phút với 1 triệu khách. Một đoạn code khác chỉ mất 0.001s ngay cả với 1 triệu khách. Vì sao chênh lệch khủng khiếp như vậy?
Question Làm sao dự đoán một thuật toán có nhanh không, mà không cần chạy thử trên dữ liệu lớn? Có cách nào "đo" độ phức tạp một cách phổ quát?
Answer Big-O notation — ngôn ngữ chuẩn của ngành để mô tả tốc độ tăng của thời gian chạy theo kích thước dữ liệu. Học Big-O = học "siêu năng lực" nhìn ra code chậm trước khi chạy nó.

III. KHỞI ĐỘNG TỪ "CÁI BIẾT"

3 ẩn dụ đời thường:

  1. Tìm số trong danh bạ. Em có 1000 số điện thoại. Tìm số của Lan: nếu danh bạ chưa sắp xếp, em đọc từng cái — O(n). Nếu đã sắp xếp theo bảng chữ cái, em mở giữa rồi loại nửa — O(log n). (Chính là bisection của Buổi 2!)
  2. Soi xe biển số. Trong bãi 100 xe, tìm xe biển 51F-99999: nếu bãi sắp xếp, nhanh; nếu lộn xộn, phải đi từng hàng.
  3. Game đoán số. Buổi 2 ta đã chơi rồi. Đoán số 1-1000 chỉ cần ~10 lần — O(log n). Đó là sức mạnh của log!

Kết nối các buổi trước: Buổi 2 dạy bisection, Buổi 4 đã thấy fib(40) chậm bất thường — bây giờ ta giải thích vì sao.

Tiết 1 (0:00 – 1:15) — Vì sao cần đo độ hiệu quả?

Hoạt động 1.1 (15 phút) — Demo "shock and awe"

  • Code 1 — Tìm trong list (chậm): ```python import time

ds = list(range(1, 10_000_001)) # 10 triệu số

start = time.time() ket_qua = 9_999_999 in ds # tìm gần cuối print(time.time() - start, "giây") - **Code 2 — Tìm trong set (nhanh):**python ds_set = set(ds) start = time.time() ket_qua = 9_999_999 in ds_set print(time.time() - start, "giây") ``` - Kết quả thường thấy: List ~0.2 giây, set ~0.000001 giây. Chênh khoảng 100,000 lần. - Câu hỏi cho SV: "Hai đoạn code làm cùng một việc. Vì sao chênh thế?" → Đó là chủ đề buổi học.

Hoạt động 1.2 (15 phút) — "Đếm phép tính"

  • Khái niệm: Big-O không đo bằng giây mà đo bằng số phép tính cơ bản theo n.
  • Demo đếm: python def vi_du_1(n): print(n) # 1 phép print(n + 1) # 1 phép print(n * 2) # 1 phép # → tổng 3 phép, không phụ thuộc n → O(1), gọi là hằng số.

python def vi_du_2(n): for i in range(n): print(i) # 1 phép, lặp n lần # → n phép → O(n), gọi là tuyến tính.

Hoạt động 1.3 (25 phút) — Bảng các Big-O thường gặp

  • Vẽ bảng to trên bảng:
Big-O Tên Ví dụ thuật toán
O(1) Hằng số Truy cập ds[5], set.add()
O(log n) Logarit Tìm kiếm nhị phân (bisection)
O(n) Tuyến tính Duyệt list để tìm
O(n log n) Tuyến-log Sắp xếp hiệu quả (sort, mergesort)
O(n²) Bình phương 2 vòng lặp lồng (sắp xếp đơn giản)
O(2ⁿ) Lũy thừa Fibonacci đệ quy ngây thơ
  • Bảng so sánh con số thực tế — cực kỳ quan trọng để gây ấn tượng:
n O(1) O(log n) O(n) O(n²) O(2ⁿ)
10 1 3 10 100 1,024
100 1 7 100 10,000 gần như vô hạn
1,000 1 10 1,000 1,000,000 bất khả thi
1,000,000 1 20 1,000,000 10¹² (nhiều ngày) bất khả thi

"Em thấy không, O(2ⁿ) chỉ chịu được n=20-30 trở xuống."

Hoạt động 1.4 (20 phút) — Quy tắc đếm Big-O

  • 3 quy tắc đơn giản:
  • Bỏ hằng số: O(3n) = O(n), O(5) = O(1).
  • Giữ bậc cao nhất: O(n² + n) = O(n²).

  • Vòng lặp lồng = nhân: for...for n×n lần → O(n²).

  • Bài tập đếm tại lớp: ```python # 1 for i in range(n): print(i) # → O(?)

# 2 for i in range(n): for j in range(n): print(i, j) # → O(?)

# 3 for i in range(100): print(i) # → O(?)

# 4 for i in range(n): print(i) for j in range(n): print(j) # → O(?) ``` Đáp án: 1) O(n), 2) O(n²), 3) O(1), 4) O(n) — không phải O(2n).

Tiết 2 (1:15 – 2:30) — So sánh thuật toán & cấu trúc dữ liệu

Hoạt động 2.1 (25 phút) — Tìm kiếm tuyến tính vs nhị phân

  • Code tuyến tính (linear search): python def tim_tuyen_tinh(ds, x): for i in range(len(ds)): if ds[i] == x: return i return -1 → Trường hợp xấu nhất: kiểm tra cả n phần tử → O(n).

  • Code nhị phân (binary search): (Yêu cầu ds đã sắp xếp.) python def tim_nhi_phan(ds, x): thap, cao = 0, len(ds) - 1 while thap <= cao: giua = (thap + cao) // 2 if ds[giua] == x: return giua elif ds[giua] < x: thap = giua + 1 else: cao = giua - 1 return -1 → Mỗi lần cắt nửa → log₂(n) lần → O(log n).

  • Demo so sánh thực tế: Cho ds 10 triệu phần tử đã sắp xếp, đo thời gian 2 cách → log n ăn đứt.

Hoạt động 2.2 (20 phút) — list vs dict vs set

  • Bảng tổng hợp (ghi nhớ cốt lõi):
Thao tác list dict set
Truy cập theo index/key O(1) O(1) —
Tìm kiếm x in s O(n) O(1) O(1)
Thêm phần tử O(1) cuối O(1) O(1)
Xóa O(n) O(1) O(1)
  • Khoảnh khắc click: "Đây chính là lý do code 2 ở đầu buổi nhanh hơn 100,000 lần — vì set tìm kiếm O(1)!"

Hoạt động 2.3 (20 phút) — Bài toán Fibonacci: giải bí ẩn của Buổi 4

  • Code fib đệ quy của Buổi 4: python def fib(n): if n <= 1: return n return fib(n-1) + fib(n-2)
  • Vẽ cây gọi đệ quy trên bảng cho fib(5): fib(5) → fib(4) + fib(3) → (fib(3) + fib(2)) + (fib(2) + fib(1)) → ... Cây phình lên rất nhanh — số nút ≈ 2ⁿ.
  • Khoảnh khắc: "Đây là lý do fib(40) chậm — phải tính 2⁴⁰ ≈ 10¹² nút."
  • Giải pháp dùng vòng lặp: python def fib_loop(n): a, b = 0, 1 for _ in range(n): a, b = b, a + b return a → O(n). fib_loop(1000) chạy gần như tức thì.

Hoạt động 3.2 (35 phút) — Lab tối ưu hóa thực tế

Hoạt động 3.3 (20 phút) — Trade-off: thời gian vs bộ nhớ

  • Khái niệm: Đôi khi tăng tốc đổi bằng tốn thêm bộ nhớ.
  • Set tìm kiếm O(1) nhưng tốn ~2-3x bộ nhớ so với list.
  • Cache (memoization) lưu kết quả → nhanh hơn nhưng tốn RAM.
  • Demo memoization cho fib: python cache = {} def fib_cache(n): if n in cache: return cache[n] if n <= 1: cache[n] = n else: cache[n] = fib_cache(n-1) + fib_cache(n-2) return cache[n] → Đệ quy có cache trở thành O(n).
  • Bài học: Lập trình giỏi = biết trade-off cho từng tình huống.

VII. 5 CÂU TRẮC NGHIỆM CUỐI BUỔI

Câu 1. Big-O của đoạn code sau?

for i in range(n):
    for j in range(n):
        print(i + j)

A. O(n) B. O(n²) C. O(2n) D. O(log n) Đáp án: B

Câu 2. Thao tác nào sau đây có Big-O lớn nhất (chậm nhất)? A. x = ds[5] (list) B. x in s (set) C. x in ds (list n phần tử) D. d["key"] (dict) Đáp án: C — list tìm kiếm O(n), các thao tác kia O(1)

Câu 3. Big-O của tìm kiếm nhị phân (binary search) trên list n phần tử đã sắp xếp? A. O(1) B. O(log n) C. O(n) D. O(n²) Đáp án: B

Câu 4. Cho n = 1,000,000, thuật toán nào không khả thi trong vài giây? A. O(n) B. O(n log n) C. O(n²) D. O(log n) Đáp án: C — n² = 10¹² phép, nhiều phút đến nhiều giờ

Câu 5. Đoạn code sau có Big-O bao nhiêu?

def f(n):
    tong = 0
    for i in range(n):
        tong = tong + i
    for j in range(n):
        tong = tong + j
    return tong

A. O(n) B. O(n²) C. O(2n²) D. O(2) Đáp án: A — 2 vòng tuần tự = O(n + n) = O(2n) = O(n)

VIII. LƯU Ý CHO GIẢNG VIÊN

  1. Đừng dạy Big-O bằng toán formal. SV không chuyên không cần định nghĩa epsilon-N. Dạy bằng trực giác con số (bảng so sánh n vs phép tính).
  2. Demo sống là vũ khí quan trọng nhất. Đo thực tế gây ấn tượng hơn nhiều slide lý thuyết. Phải có Jupyter mở từ đầu.
  3. Quy tắc đếm 3 dòng (bỏ hằng, giữ bậc cao, lồng = nhân) là tất cả những gì cần. Đừng dạy thêm Θ, Ω, o, ω.
